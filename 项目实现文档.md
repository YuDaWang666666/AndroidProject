一、实训任务原文档内容
================================================================

1. 项目概述
   实现新蜂商城的 Android 客户端。
   接口地址：http://115.158.64.84:28019/swagger-ui/index.html
   App 参考：http://115.158.64.84
   原项目地址：
     后端 API：https://gitee.com/hbxy/newbee-mall-api
     Vue 前端：https://gitee.com/newbee-ltd/newbee-mall-vue3-app
   参考 Android 项目：https://gitee.com/android-zz/ShopApp.git

2. 成绩规则
   - 完成两项任务，成绩 80 分
   - 由实训老师根据完成情况加分或扣分
   - 可通过完成额外功能获得加分

3. 接口使用说明
   - 登录接口：POST /api/v1/user/login
     请求体：{"loginName":"16666666666","password":"<md5加密后的密码>"}
     密码需 MD5 加密（32位小写），示例：密码 88888888
     登录成功后返回的 data 字段即为 token
   - Token 使用方法：
     SharedPreferences info = getSharedPreferences("info", MODE_PRIVATE);
     String token = info.getString("token","");
     conn.setRequestProperty("token", token);
   - 图片链接处理：如果没有主机名，校内加 http://115.158.64.84:28019，校外加 http://47.99.134.126:28019
   - 所有接口均为 HTTP 协议（非 HTTPS）
   - AndroidManifest 需要设置 android:usesCleartextTraffic="true"

4. 技术要求
   - 网络访问请使用 HttpURLConnection，不要使用第三方网络库
   - 轮播图组件：implementation("io.github.youth5201314:banner:2.2.3")
   - 图片加载：implementation("com.github.bumptech.glide:glue:4.12.0")
   - Gradle 下载请使用镜像：mirrors.aliyun.com/gradle/distributions/v8.11.1/gradle-8.11.1-all.zip
   - 由于登录和注册功能已经提供，再实现是没有加分的

二、功能模块详细设计
================================================================

=== 模块 1：主界面 MainActivity ===

底部导航栏使用 BottomNavigationView，包含 4 个 Tab：
  - 首页 (HomeFragment)
  - 分类 (CategoryFragment)
  - 购物车 (CartFragment)
  - 我的 (MyFragment)

实现要点：
  - 使用 FrameLayout + Fragment 替换实现页面切换
  - BottomNavigationView 的 menu 资源定义 4 个 item
  - 每次切换 Tab 时 replace Fragment
  - MainActivity 提供 switchToCategory() 和 switchToCart() 方法供其他页面调用

=== 模块 2：HomeFragment 首页 ===

2.1 标题栏
  - 浮动在页面顶部，初始透明
  - 包含搜索框（点击跳转 SearchActivity）和登录图标
  - 向下滑动时逐渐变为白色背景（通过 ScrollView 的 OnScrollChangeListener 实现）
  - 搜索框文字默认显示"山河无恙，人间皆安"

2.2 轮播图 Banner
  - 使用 Banner 第三方组件
  - API: GET /api/v1/index/carousels
  - 返回数据格式：{"data":[{"carouselUrl":"/files/xxx.jpg",...}]}
  - 使用 Glide 加载图片
  - 如果 URL 不以 http 开头，需要拼接 BASE_URL

2.3 导航栏 GridView
  - 使用 GridView 控件，4列
  - 8 个固定图标：新蜂超市、新蜂服饰、母婴生活、图书音像、充值缴费、生鲜到家、家具家装、更多分类
  - 图标需要从 Vue 版 App 保存到 drawable 资源
  - "更多分类"点击后切换到 CategoryFragment

2.4 商品列表（新品上线/热门商品/最新推荐）
  - API: GET /api/v1/index/newGoods
  - API: GET /api/v1/index/hotGoods
  - API: GET /api/v1/index/recommendGoods
  - 使用 GridView 控件，2列
  - 显示商品封面图、名称、价格
  - 点击商品跳转 DetailActivity

=== 模块 3：CategoryFragment 分类 ===

3.1 左侧分类列表
  - 使用 ListView 控件
  - API: GET /api/v1/categories
  - 返回数据：一级分类列表
  - 选中项高亮显示

3.2 右侧商品列表
  - 使用 MyGridView 控件（自定义 GridView，自动适配高度）
  - 点击左侧分类后，根据分类 ID 查询商品
  - API: GET /api/v1/search?pageNumber=1&goodsCategoryId={id}&keyword=&orderBy=
  - 显示商品封面图、名称、价格
  - 点击商品跳转 DetailActivity

3.3 MyGridView 自定义控件
  继承 GridView，重写 onMeasure 方法：
  @Override
  protected void onMeasure(int widthMeasureSpec, int heightMeasureSpec) {
      int expandSpec = MeasureSpec.makeMeasureSpec(Integer.MAX_VALUE >> 2, MeasureSpec.AT_MOST);
      super.onMeasure(widthMeasureSpec, expandSpec);
  }
  目的：使 GridView 可以完全展开，不会被 ScrollView 截断。

=== 模块 4：CartFragment 购物车 ===

4.1 功能要求
  - 展示当前登录用户的购物车商品列表
  - 每项包含：复选框、商品图片、名称、价格、数量（加减按钮）、删除按钮
  - 底部显示合计金额和"去结算"按钮
  - 未登录时显示空购物车提示
  - 进入页面时自动刷新数据

4.2 API 接口
  - 获取购物车列表：GET /api/v1/shop-cart (需要 token)
  - 修改数量：PUT /api/v1/shop-cart (需要 token)
    请求体：{"cartItemId":1,"goodsCount":2}
  - 删除商品：DELETE /api/v1/shop-cart/{cartItemId} (需要 token)

4.3 实现要点
  - 使用 RecyclerView 展示列表
  - CheckBox 的选中状态控制合计金额
  - 数量修改后同步更新服务端
  - 删除后刷新列表

=== 模块 5：MyFragment 我的 ===

5.1 功能要求
  - 顶部显示用户头像和用户名
  - 已登录：显示用户名和"退出登录"按钮
  - 未登录：显示"未登录"和"登录/注册"按钮
  - 功能菜单：我的订单、地址管理
  - 底部显示开发小组及人员介绍（建议修改为实际团队信息）

5.2 实现要点
  - 使用 SharedPreferences 判断登录状态
  - 菜单项点击前判断登录状态
  - 退出登录清除 SharedPreferences 中的 token

=== 模块 6：LoginActivity 登录 ===

6.1 功能要求
  - 手机号 + 密码登录
  - 密码输入后需 MD5 加密再发送
  - 登录成功后保存 token 到 SharedPreferences
  - 可以和注册做在同一个界面，或提供注册入口

6.2 API 接口
  - 登录：POST /api/v1/user/login
    请求体：{"loginName":"手机号","password":"md5加密密码"}
    返回：{"resultCode":200,"data":"token字符串"}
  - 注册：POST /api/v1/user/register
    请求体：{"loginName":"手机号","password":"md5加密密码"}

6.3 MD5 加工工具类
  public static String md5(String input) {
      MessageDigest md = MessageDigest.getInstance("MD5");
      byte[] digest = md.digest(input.getBytes());
      StringBuilder sb = new StringBuilder();
      for (byte b : digest) {
          sb.append(String.format("%02x", b & 0xff));
      }
      return sb.toString();
  }

=== 模块 7：SearchActivity 商品搜索 ===

7.1 功能要求
  - 顶部搜索输入框 + 搜索按钮
  - 中部 TabLayout 显示排序选项：推荐、新品、价格
  - 下方 RecyclerView 显示搜索结果（2列网格）
  - 点击商品跳转详情页

7.2 API 接口
  - 推荐排序：GET /api/v1/search?pageNumber=1&keyword=xxx&orderBy=
  - 新品排序：GET /api/v1/search?pageNumber=1&keyword=xxx&orderBy=new
  - 价格排序：GET /api/v1/search?pageNumber=1&keyword=xxx&orderBy=price
  - 分类搜索：GET /api/v1/search?pageNumber=1&goodsCategoryId=xxx&keyword=&orderBy=

7.3 实现要点
  - TabLayout 的 onTabSelected 中切换 orderBy 参数
  - 支持从 CategoryFragment 传入 categoryId 进行分类搜索
  - 搜索结果为空时给出提示

=== 模块 8：DetailActivity 商品详情 ===

8.1 功能要求
  - 顶部商品大图（轮播或首张图）
  - 商品名称、简介
  - 当前价格（红色）、原价（灰色删除线）
  - 数量选择器（加减按钮）
  - 底部操作栏：加入购物车、立即购买
  - 右上角购物车图标

8.2 API 接口
  - 商品详情：GET /api/v1/goods/detail/{goodsId}
  - 添加购物车：POST /api/v1/shop-cart (需要 token)
    请求体：{"goodsId":1,"goodsCount":1}

8.3 实现要点
  - 加入购物车前检查登录状态
  - 立即购买 = 加入购物车 + 跳转购物车页面
  - 使用 Glide 加载商品图片

=== 模块 9：地址管理 ===

9.1 AddressListActivity 地址列表
  - 使用 RecyclerView 展示地址列表
  - 每项显示：收件人、手机号、详细地址、默认标识
  - 支持新增地址和设为默认地址
  - 支持选择模式（从订单确认页跳转时使用）

9.2 AddressEditActivity 地址编辑
  - 输入项：收件人、手机号、省市区（可简化为手动输入）、详细地址
  - 默认地址开关
  - 支持新增和编辑两种模式
  - 建议使用第三方省市区联动组件

9.3 API 接口
  - 地址列表：GET /api/v1/address (需要 token)
  - 新增地址：POST /api/v1/address (需要 token)
  - 更新地址：PUT /api/v1/address (需要 token)
    请求体：{"addressId":1,"userName":"xxx","userPhone":"xxx","provinceName":"xxx","cityName":"xxx","regionName":"xxx","detailAddress":"xxx","defaultFlag":0}

=== 模块 10：订单管理 ===

10.1 OrderListActivity 订单列表
  - 顶部 TabLayout 显示订单状态筛选：
    全部、待付款(0)、待确认(1)、待发货(2)、已发货(3)、交易完成(4)
  - 使用 RecyclerView 展示订单列表
  - 每个订单卡片：订单号、状态、商品列表（嵌套 RecyclerView）、总价、创建时间
  - 点击订单跳转订单详情

10.2 OrderDetailActivity 订单详情
  - 显示订单号、状态、创建时间、总价
  - 商品列表
  - 操作按钮：待付款->去付款，已发货->确认收货

10.3 OrderConfirmActivity 订单确认
  - 收货地址选择（点击跳转地址列表选择模式）
  - 提交订单按钮
  - 无地址时提示先添加地址

10.4 PayActivity 支付页面
  - 显示订单号
  - 支付方式选择（支付宝 payType=1 / 微信 payType=2）
  - 确认支付按钮（模拟支付）

10.5 API 接口
  - 订单列表：GET /api/v1/order?pageNumber=1&status={status} (需要 token)
  - 订单详情：GET /api/v1/order/{orderId} (需要 token)
  - 创建订单：POST /api/v1/saveOrder (需要 token)
    请求体：{"addressId":1,"cartItemIds":[]}
  - 支付：GET /api/v1/paySuccess?orderNo=xxx&payType=1 (需要 token)
  - 确认收货：PUT /api/v1/order/finish/{orderId} (需要 token)

六、关键实现代码片段
================================================================

6.1 HttpUtil 网络请求封装
-------------------------------------------------------
支持 GET/POST/PUT/DELETE 四种方法，自动处理 token 注入。

6.2 MD5 加密
-------------------------------------------------------
java.security.MessageDigest 获取 MD5 实例，将 byte 数组转为 32 位小写十六进制字符串。

6.3 Token 管理
-------------------------------------------------------
使用 SharedPreferences 存储，key 为 "info"，包含 token、userName、userId 三个字段。

6.4 自定义 MyGridView
-------------------------------------------------------
重写 onMeasure，使用 Integer.MAX_VALUE >> 2 作为 MeasureSpec，使 GridView 在 ScrollView 中完全展开。

七、开发注意事项
================================================================

1. 网络请求必须在子线程执行，UI 更新必须回到主线程 (runOnUiThread)
2. HTTP 请求需要在 AndroidManifest 中设置 android:usesCleartextTraffic="true"
3. 需要添加 INTERNET 权限
4. 图片 URL 如果不以 http 开头，需要拼接 BASE_URL
5. 从 Vue 版 App 保存导航图标等资源图片
6. 密码传输前必须进行 MD5 加密
7. 价格单位为分，显示时需要除以 100 或直接使用整数显示
8. TabLayout 需要添加 app 依赖：com.google.android.material:material
